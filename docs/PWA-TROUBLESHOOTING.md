# تنظيف بيانات إيجار من Chrome (دون فقد سجلّك)

إذا رفض Chrome على هاتفك تثبيت إيجار كتطبيق مستقلّ (يكتفي باختصار)،
فالمصدر شبه دائماً **service worker قديم** أو **manifest قديم** عالقان
في تخزين Chrome للموقع. هنا 3 طرق مرتّبة من الأسهل للأقوى — كلها تحذف
بيانات **إيجار فقط** ولا تمسّ مواقعك الأخرى أو سجلّ التصفّح أو كلمات
المرور.

---

## الطريقة 1 — صفحة `/reset.html` (أسهل، الموصى بها)

1. افتح Chrome على هاتفك.
2. اكتب في شريط العنوان: `https://<عنوان-إيجار-عندك>/reset.html`
   مثلاً: `https://ejar.runasp.net/reset.html`
3. اضغط زرّ «**امسح وأعد التحميل**».
4. انتظر ثانيتين — يعيد التحميل تلقائياً على الصفحة الرئيسية بحالة
   نظيفة تماماً.
5. تفاعل مع التطبيق ~30 ثانية → تظهر أيقونة التثبيت ⊕ في شريط العنوان.
6. اضغطها → اختر «**تثبيت**» → ستحصل على تطبيق مستقلّ بنافذته الخاصة.

---

## الطريقة 2 — من إعدادات Chrome للموقع فقط

إذا لم تستطع فتح `/reset.html` لسبب ما:

1. افتح Chrome → اضغط على شريط العنوان → اكتب `chrome://settings/content/all`
2. ابحث عن `ejar` أو نطاقك (مثل `ejar.runasp.net`).
3. اضغط على البند → اضغط «**حذف وإعادة الضبط**» / «Clear & reset».
4. هذا يمسح **هذا الموقع فقط**: cookies، storage، service worker،
   permissions. لا يمسّ مواقع أخرى ولا سجلّ التصفّح.
5. عُد إلى التطبيق وأعد التحميل.

طريق مختصر آخر:
- في صفحة إيجار، اضغط القفل/المعلومة في شريط العنوان (يسار الـ URL على
  أندرويد) → «إعدادات الموقع» → «حذف وإعادة الضبط».

---

## الطريقة 3 — DevTools على Chrome سطح المكتب (للمطوّر)

1. افتح إيجار على Chrome سطح المكتب.
2. افتح DevTools بـ `F12` → تبويب **Application**.
3. على اليسار → **Storage** → اضغط **Clear site data** (مع تفعيل خانة
   «Including third-party cookies» إن ظهرت).
4. لو أردت فقط SW → **Service Workers** → اضغط «Unregister» للنطاق.
5. أعد التحميل (`Ctrl+Shift+R` لإجبار bypass cache).

---

## آلية التحديث التلقائي بعد كل نشر

ابتداءً من النسخة الحاليّة، أصبح التطبيق يفرض تنظيفاً تلقائياً عند كلّ
نشر جديد بدون تدخّل المستخدم. الآلية:

1. عند كل بدء تشغيل، يجلب التطبيق `version.json` بـ `cache: 'no-store'`
   — يتجاوز كل cache.
2. يقارن مع آخر نسخة محفوظة في `localStorage`.
3. لو اختلفت → يُلغي تسجيل كل service worker، يحذف كل caches، ثم يعيد
   التحميل بـ `?ac_v=<new>` (يكسر cache CDN/proxy).
4. شاشة «جاري تحديث إيجار…» الزرقاء تظهر للمستخدم لثوانٍ ثم يفتح
   التطبيق على النسخة الجديدة مباشرة.

### كيف ترفع نسخة عند نشر جديد (للمطوّر)

عدّل الملفّين:

**`wwwroot/version.json`**:
```json
{ "version": "2026.04.27.2", "released_at": "2026-04-27" }
```

**`wwwroot/service-worker.js`** — السطر الأوّل:
```js
const VERSION = 'ejar-pwa-v5-2026-04-27';
```

(الرقمان لا يجب أن يتطابقا — أيّ تغيير في `version` كافٍ ليُجبر العميل
على المسح + إعادة التحميل.)

ادفع وانشر — كلّ جهاز فُتح فيه التطبيق سابقاً سيتحدّث تلقائياً المرّة
القادمة. لا حاجة لأن يفعل المستخدم شيء.

---

## ملاحظات

- **بعد التنظيف ستفقد جلسة الدخول الحالية** — التطبيق سيطلب رمز التحقّق
  مجدّداً. التفضيلات (المدينة، اللغة، الأيقونة الداكنة) قد تُحفظ أو
  تُمسح حسب طريقة التنظيف.
- على iOS/Safari لا توجد آليّة `service worker` بنفس قوّة Chrome، لذلك
  هذه المشكلات أقل شيوعاً عليه.
- إذا فشل كل ما سبق، آخر حلّ: **حذف اختصار/تطبيق إيجار من الشاشة
  الرئيسية** (long-press → uninstall) ثم إعادة التثبيت من Chrome.
